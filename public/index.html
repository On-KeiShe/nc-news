Northcoders NC News API

Welcome to the Northcoders News API, a RESTful API providing access to a database of articles, topics, users, and comments. All responses are in JSON format.

GET /api

Returns a JSON object describing all available endpoints, including methods and descriptions.

Route Parameter: None

Query Parameters: None

Request Body: Not required

Example Response:

{
  "endpoints": {
    "GET /api/topics": "Returns all topics",
    "GET /api/articles": "Returns all articles",
    "GET /api/articles/:article_id": "Returns a single article by ID",
    "PATCH /api/articles/:article_id": "Updates the vote count for an article",
    "GET /api/articles/:article_id/comments": "Returns comments for a given article",
    "POST /api/articles/:article_id/comments": "Adds a new comment to an article",
    "DELETE /api/comments/:comment_id": "Deletes a comment by ID",
    "GET /api/users": "Returns all users"
  }
}

GET /api/topics

Returns a list of all topics available on the site.

Route Parameter: None

Query Parameters: None

Request Body: Not required

Example Response:

[
  { "slug": "coding", "description": "Code is love, code is life" },
  { "slug": "football", "description": "Footie news and opinions" },
  { "slug": "cooking", "description": "Delicious recipes and cooking tips" }
]

GET /api/articles

Returns all articles. You can filter, sort, and order them using query parameters.

Route Parameter: None

Query Parameters:

  • Filter by topic (e.g. ?topic=coding)
  • Sort by column (e.g. ?sort_by=votes)
  • Order ascending/descending (e.g. ?order=asc)

Request Body: Not required

Example Response:

[
  { "article_id": 1, "title": "The Rise of Node.js", "topic": "coding", "author": "jessjelly", "votes": 100, "comment_count": 13 },
  { "article_id": 2, "title": "Football Tactics in 2025", "topic": "football", "author": "grumpy19", "votes": 42, "comment_count": 5 }
]

GET /api/articles/:article_id

Returns full details of a single article by article_id.

Route Parameter: Article ID (e.g. /api/articles/1)

Query Parameters: None

Request Body: Not required

Example Response:

{
  "article_id": 1,
  "title": "The Rise of Node.js",
  "topic": "coding",
  "author": "jessjelly",
  "body": "Node.js has taken the backend world by storm...",
  "created_at": "2020-07-09T20:11:00.000Z",
  "votes": 100,
  "comment_count": 13
}

PATCH /api/articles/:article_id

Update the vote count of a specific article.

Route Parameter: Article ID (e.g. /api/articles/1)

Request Body:

{
  "inc_votes": 1
}

Example Response:

{
  "article_id": 1,
  "votes": 101
}

GET /api/articles/:article_id/comments

Returns all comments for the specified article.

Route Parameter: Article ID (e.g. /api/articles/1/comments)

Request Body: Not required

Example Response:

[
  { "comment_id": 1, "author": "grumpy19", "body": "Great article!", "votes": 5, "created_at": "2020-07-09T20:11:00.000Z" }
]

POST /api/articles/:article_id/comments

Add a new comment to an article.

Route Parameter: Article ID (e.g. /api/articles/1/comments)

Request Body:

{
  "username": "grumpy19",
  "body": "This is my new comment!"
}

Example Response:

{
  "comment_id": 2,
  "author": "grumpy19",
  "body": "This is my new comment!",
  "votes": 0,
  "created_at": "2025-09-30T20:00:00.000Z"
}

DELETE /api/comments/:comment_id

Deletes a comment by its ID.

Route Parameter: Comment ID (e.g. /api/comments/1)

Request Body: Not required

Example Response:

{
  "msg": "Comment deleted"
}

GET /api/users

Returns a list of all users.

Route Parameter: None

Request Body: Not required

Example Response:

[
  { "username": "jessjelly", "name": "Jess Jelly", "avatar_url": "https://example.com/avatar1.jpg" },
